# Assets Spec — Beija

All visual assets required for App Store + Google Play + PWA distribution. None of the PNGs below exist yet (placeholders are 192/512 px solid pink squares). This doc is the source of truth for what designers / build pipelines must produce before submission.

---

## 1. App Icon Master — 1024 × 1024

The single source from which every other size is derived.

| Field | Value |
|---|---|
| Format | PNG, sRGB, 8-bit, **no transparency** |
| Background | Solid (Apple rejects transparent icons) |
| Corner radius | **None** — Apple/Android round it themselves |
| Safe area | Keep all design inside the centered 80% (≈ 819 × 819) |
| Filename (output) | `frontend/public/icons/icon-1024.png` |

**Design direction** (see `ICON_DESIGN.md` for full spec):

- Background: linear gradient pink-coral, top-left `#FF6B9D` → bottom-right `#FFA07A`.
- Foreground: stylized letter **B** in white (`#FFFFFF`), centered, ~60% of safe area in height.
- No photos, no faces, no small text (gets unreadable at 40 px).
- Subtle inner glow / shadow OK if it survives down-scaling.

## 5. App Store screenshots

Apple requires **at least one device class**; we'll ship two for premium feel, plus the legacy 5.5" to catch older audits.

| Display | Size (portrait) | Required for App Store? |
|---|---|---|
| 6.7" / 6.9" iPhone | **1290 × 2796** | yes (modern phones) |
| 6.5" iPhone | **1242 × 2688** | yes (recommended) |
| 5.5" iPhone | **1242 × 2208** | optional, accepted |
| iPad 12.9" | 2048 × 2732 | only if app supports iPad |

**Composition:** 5 screenshots minimum, 10 max per locale (PT-BR primary, EN-US secondary if enabling). Each screenshot is a captured screen + an overlay headline in brand font.

| # | Theme | Screen captured | Overlay headline (PT-BR) |
|---|---|---|---|
| 1 | Hero swipe deck | `/discover` with top card visible, NOPE peek on the left | "Conheça gente perto." |
| 2 | Match modal | Match modal with confetti + 2 photos + textarea pre-filled | "É beijo na boca!" |
| 3 | Chat realtime | Chat with one received + one sent message, typing indicator visible | "Converse sem complicação." |
| 4 | Profile setup | `/profile` with 6-slot photo grid + bio + sliders | "Você no controle." |
| 5 | Onboarding speed | Side-by-side step 1 + step 2 collage with "5 toques" headline | "Conta criada em 60 segundos." |

Optional 6th: report/safety screen with the moderation modal mid-flow.

## 6. App preview video (optional but recommended)

Per Apple guidelines:

| Field | Value |
|---|---|
| Duration | 15–30 s |
| Resolution | match the device class screenshot (e.g. 886 × 1920 px portrait for 6.5") |
| Format | H.264 MP4, ProRes 422 HQ acceptable |
| Frame rate | 30 fps |
| Audio | optional but recommended; no licensed music |
| Localized | per-locale, PT-BR primary |

**Suggested storyboard (15 s):**
1. (0:00–0:02) Logo splash 1.5s, fades to /signin
2. (0:02–0:04) Tap "Continuar com Apple" → fast-forwards through onboarding (collage)
3. (0:04–0:08) Discover deck: 2 swipes left, 1 swipe right
4. (0:08–0:11) Match modal with confetti
5. (0:11–0:14) Tap "Enviar mensagem", arrives at chat, typing indicator flashes
6. (0:14–0:15) End card: logo + "Beija — beija.app"

## 7. Generation pipeline

Once the 1024 master and a single 2880×2880 splash master exist, run:

```bash
cd frontend
npx @capacitor/assets generate --iconBackgroundColor "#FF4D88" --iconBackgroundColorDark "#1c0a2b"
```

This populates `ios/App/App/Assets.xcassets/` and `android/app/src/main/res/`. The output is automatically picked up by `npx cap sync`.
